# PDCA Cycle

## When to Use
- Before implementing any significant change
- When continuous improvement is goal
- Learning from successes and failures
- Reducing trial-and-error waste

## 4-Phase Process

### Plan
- Generate hypothesis about approach
- Predict expected outcomes
- Identify success criteria
- Document assumptions

### Do
- Execute with checkpoint tracking
- Collect data during execution
- Note deviations from plan

### Check
- Evaluate results against predictions
- Use reflection tools (think_about_*)
- Identify what worked / didn't work
- Analyze root causes of deviations

### Act
- Extract improvements and lessons
- Update approach based on learning
- Document patterns for reuse
- Implement prevention measures

## Mandatory Elements
- Hypothesis BEFORE execution
- Prediction vs. actual comparison
- Root cause analysis for deviations
- Documentation of lessons learned
