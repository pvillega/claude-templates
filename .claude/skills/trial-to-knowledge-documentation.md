# Trial-to-Knowledge Documentation

## When to Use
- Experimenting with new approaches
- Learning from successes and failures
- Building knowledge base over time
- Capturing trial-and-error lessons

## Three-Tier System

### Tier 1: temp/ (Experimentation)
- Hypothesis, experiment notes, raw lessons
- Marked with 試行錯誤 (trial and error)
- 7-day lifecycle before cleanup
- Fast capture during exploration

### Tier 2: patterns/ (Success)
- Formalized successful approaches
- Marked with 清書 (clean copy/fair copy)
- Evolved from temp/ when approach validates
- Reusable patterns for future work

### Tier 3: mistakes/ (Failure Prevention)
- What happened, root cause, why missed
- Fix applied, prevention checklist
- Lesson learned
- Prevents repeating errors

## Evolution Rules
- temp/ → patterns/ when approach succeeds repeatedly
- temp/ → mistakes/ when approach fails with lessons
- temp/ → deleted after 7 days if no evolution

## Quality Standards
- Capture IMMEDIATELY while context fresh
- Include enough detail for future use
- Prevention checklists actionable
- Patterns reusable by others
