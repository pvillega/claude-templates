# Requirements-Driven PRD Creation

## When to Use
- Requirements discovery complete
- Need formal documentation
- Preparing for implementation
- Stakeholder alignment required

## PRD Template Structure
1. Executive Summary
2. Problem Statement
3. User Stories
4. Requirements (Functional/Non-functional)
5. Acceptance Criteria
6. Success Metrics
7. Priority Rankings
8. Out of Scope
9. Dependencies
10. Risks and Mitigations
