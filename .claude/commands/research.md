Conduct exhaustive research and analysis on any topic, concept, or question using the comprehensive-researcher agent. This command gathers information from multiple sources, explores different perspectives, identifies tradeoffs, and provides comprehensive reports to empower informed decision-making.

## When to Use

- Before making technical architecture decisions
- When evaluating competing solutions or technologies
- To understand complex topics with multiple considerations
- For comparative analysis between different approaches
- When you need exhaustive information before planning
- To explore tradeoffs and alternatives systematically
- For research that requires multiple perspectives and angles

## Examples

```
/research What are the best approaches for implementing real-time collaboration in web applications?
/research Compare PostgreSQL vs MongoDB for e-commerce applications
/research Modern authentication patterns and security considerations for SaaS applications
/research State management solutions in React: comprehensive comparison
/research Microservices vs monolith architecture tradeoffs
```

## Process

The command launches the comprehensive-researcher agent which handles all research execution using its comprehensive methodology:

```
Task: comprehensive-researcher
Description: "Comprehensive research on [topic]"
Prompt: "Conduct exhaustive research and analysis on: [user's query]"
```

The agent autonomously executes its full research methodology, including query processing, information gathering, multi-angle analysis, alternative exploration, and report generation. All output formatting and file management is handled by the agent according to its specifications.

## Expected Outcomes

After running this command, you'll have comprehensive research stored in a structured report with complete understanding of the topic, alternative options, decision factors, and risk assessments to support informed decision-making.

## Follow-up Actions

Consider these next steps after comprehensive research:

1. **Decision Making** - Use research to make informed technology/approach choices
2. **Detailed Planning** - Create implementation plans based on chosen direction
3. **Stakeholder Review** - Share findings with relevant team members
4. **Documentation Update** - Incorporate insights into project documentation

The research provides the foundation; you make the choices based on your specific context and requirements.
